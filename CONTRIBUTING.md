# Contributing to Prometheus

First off, thank you for considering contributing to Prometheus! It's people like you that make this platform a robust and explainable quantitative research engine.

## Code of Conduct

By participating in this project, you are expected to uphold our standard of professionalism and respect. We prioritize technical honesty, robust engineering discourse, and respectful collaboration.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report for Prometheus.
- Use the GitHub issue search — check if the issue has already been reported.
- Check if the issue has been fixed — try to reproduce it using the latest `main` or development branch in the repository.
- Isolate the problem — create a reduced test case.

### Suggesting Enhancements
This section guides you through submitting an enhancement suggestion, including completely new features and minor improvements to existing functionality.
- Describe the current behavior and the behavior you expected to see.
- Explain why this enhancement would be useful.
- Provide a clear use case or architectural diagram if possible.

### Pull Requests
- Keep your pull requests focused on a single issue.
- Include appropriate documentation updates (e.g. `README.md` or inline comments).
- Ensure your code passes all established stability and robustness validation tests before submitting.
- Provide a clear, descriptive commit message.

## Development Setup
Please refer to the `Deployment & Setup` section in the `README.md` to get your local development environment up and running. 

Thank you for helping us push the boundaries of explainable, institutional-grade quantitative research!
